# Module 5: Responsible Research Use

Duration: 15 minutes

## Goal

Define responsible boundaries for using an AI agent in research work.

By the end of this module, you should have a `responsible-use.md` file that explains what you can share, what you should not share, what you can push to GitHub, and what you are responsible for verifying.

You will create:

- `responsible-use.md`

## Step 1: Inventory Your Workspace

Ask your agent to inspect your workshop workspace and list the files you have created.

Example prompt:

```text
Inventory the files in my workshop workspace. Classify each file as safe to share, review before sharing, or do not share. Do not stage, commit, delete, or upload anything.
```

## Step 2: Identify Risks

Ask your agent to identify responsible-use risks in the files and workflows.

Example prompt:

```text
Identify risks in my workspace related to credentials, private data, restricted data, copyright, unclear licenses, generated outputs, and unsupported claims. For each risk, explain what should be checked before sharing or pushing.
```

## Step 3: Define GitHub Sharing Rules

Ask your agent to draft rules for what should and should not be committed or pushed.

Example prompt:

```text
Create GitHub sharing rules for my research workspace. Include what should be committed, what should not be committed, what might need to be added to .gitignore, and what needs instructor approval before sharing.
```

## Step 4: Track Sources and Copyright

Ask your agent to add source-use rules.

Example prompt:

```text
Create source and copyright rules for this workspace. Prefer public or open sources, track links and citations, avoid copying large copyrighted passages, summarize with attribution, and keep raw-source notes separate from my own synthesis.
```

## Step 5: Write an AI Disclosure Statement

Ask your agent to draft a short disclosure statement for your research workflow.

Example prompt:

```text
Draft an AI disclosure statement for this research workflow. It should say where AI helped, what was verified by a human, and what remains uncertain.
```

## Step 6: Create `responsible-use.md`

Ask your agent to create:

```text
responsible-use.md
```

inside your workspace.

Example prompt:

```text
Review my workshop workspace for responsible research use. Inventory the files and classify each as safe to share, review before sharing, or do not share. Identify risks related to credentials, private data, restricted data, copyright, unclear licenses, generated outputs, and unsupported claims. Then create responsible-use.md with a share/push decision table, data boundaries, GitHub sharing rules, source/copyright notes, AI disclosure statement, do-not-use list, and open concerns. Do not stage, commit, delete, or upload anything.
```

## Step 7: Add a Do-Not-Use List

Add a section that clearly states when you should not use AI.

Examples:

- Do not use AI to process private or restricted data without approval.
- Do not use AI to generate citations without checking them.
- Do not use AI to make final research claims without verification.
- Do not use AI to bypass source licenses, institutional policies, or research protocols.
- Do not use AI to hide uncertainty.

## Quick Checkpoint

Be ready to show:

- One file classified as safe to share.
- One file or file type classified as review before sharing or do not share.
- One responsible-use boundary you applied.

## Completion Check

Before moving on, confirm that `responsible-use.md` includes:

- Workspace inventory summary.
- Share/push decision table.
- Data boundaries.
- GitHub sharing rules.
- Source and copyright rules.
- AI disclosure statement.
- Verification responsibilities.
- A do-not-use list.
- Open concerns.

## Key Message

AI assistance does not transfer responsibility. It increases the need for documentation.
